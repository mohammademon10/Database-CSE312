## 🧭 Overview

This repository is the complete coursework portfolio for **Database Management Systems (DBMS) — CSE312** at **Daffodil International University**. It documents an 8-week progressive journey through relational database theory and hands-on SQL programming, culminating in advanced procedural SQL development.

Every folder represents a structured learning milestone — from crafting normalized schemas to automating database logic with triggers and views. The repository also includes all lab evaluations, formal reports, and course reference materials, making it a self-contained knowledge base for anyone learning MySQL.

## 🚀 Build & Run Instructions

### Prerequisites

Ensure you have the following installed on your machine:

| Tool | Version | Download |
|:---|:---:|:---|
| MySQL Server | 8.0+ | [mysql.com/downloads](https://www.mysql.com/downloads/) |
| MySQL Workbench *(optional)* | Latest | [MySQL Workbench](https://www.mysql.com/products/workbench/) |
| Git | Latest | [git-scm.com](https://git-scm.com/) |

<br/>

### Step 1 — Clone the Repository

```bash
git clone https://github.com/mohammademon10/Database-CSE312.git
cd Database-CSE312
```

### Step 2 — Start Your MySQL Server

```bash
# On Linux / macOS
sudo service mysql start

# On Windows (run as Administrator)
net start MySQL80

# Or via XAMPP / WAMP Control Panel — click "Start" on MySQL module
```

### Step 3 — Log Into MySQL CLI

```bash
mysql -u root -p
# Enter your MySQL root password when prompted
```

### Step 4 — Set Up the Nation Database (for Weeks 3–7)

```sql
-- Inside MySQL CLI:
CREATE DATABASE nation;
USE nation;
SOURCE week_03/nation.sql;
```

### Step 5 — Execute Any Weekly Script

```sql
-- Example: Run the JOIN exercises from Week 5
USE nation;
SOURCE week-05/joins.sql;

-- Example: Run Stored Procedures from Week 6
SOURCE week-06/Stored_Procedure.sql;
CALL fetching_alldata();

-- Example: Run Triggers from Week 7
SOURCE week-07/Trigger.sql;

-- Example: Run the full University schema from Week 8
CREATE DATABASE university_db;
USE university_db;
SOURCE week-08/university.sql;
```

### Step 6 — Run Lab Evaluations

```sql
-- Lab Evaluation 1 — Library schema
CREATE DATABASE library_db;
USE library_db;
SOURCE Lab_Evaluation/Lab_Evaluation1/Library.sql;
SOURCE Lab_Evaluation/Lab_Evaluation1/Library_solution.sql;
```

> 💡 **Tip:** Use **MySQL Workbench** for a GUI experience — open any `.sql` file and press `Ctrl+Shift+Enter` to execute the full script.

### JOIN Query — Nations with Their Regions

```sql
SELECT nation.name AS Nation, nation.population AS Population, region.name AS Region
FROM nation
INNER JOIN region ON nation.region_id = region.region_id
ORDER BY nation.population DESC
LIMIT 5;
```

## 🧠 What I Learned

### 🏛️ Database Theory & Design
- Relational model fundamentals: entities, attributes, tuples, and domains
- Normalization theory: 1NF → 2NF → 3NF → BCNF — eliminating redundancy and anomalies
- ER diagram design and mapping entity relationships to relational schema
- Understanding primary keys, foreign keys, composite keys, and integrity constraints

### 🔤 SQL Language Mastery
- **DDL:** CREATE, ALTER, DROP, TRUNCATE — schema lifecycle management
- **DML:** INSERT, UPDATE, DELETE — precise data manipulation with transactional awareness
- **DQL:** SELECT with filtering, sorting, aggregation, and conditional expressions
- Complex filtering using BETWEEN, LIKE, IN, EXISTS, ANY, and ALL operators

### 🔗 Advanced Query Techniques
- Mastering all JOIN types: INNER, LEFT, RIGHT, FULL OUTER, and CROSS JOIN
- Writing correlated and non-correlated subqueries in SELECT, WHERE, and FROM clauses
- Multi-level nested queries spanning three or more related tables
- Aggregate functions: COUNT, SUM, AVG, MIN, MAX with GROUP BY and HAVING

### ⚙️ Procedural SQL Programming
- Writing reusable **Stored Procedures** with IN/OUT parameters and DELIMITER syntax
- Designing **User-Defined Functions** for encapsulated, modular SQL computation
- Implementing **Triggers** (BEFORE/AFTER INSERT/UPDATE/DELETE) using NEW and OLD virtual tables
- Building **Views** for data abstraction, query reuse, and access control

### 📋 Professional Documentation
- Structuring formal technical lab reports with schemas, queries, output screenshots, and analysis
- Version-controlling academic SQL projects using Git with meaningful commit history
- Translating real-world scenarios (hospital, library, university) into normalized database designs
